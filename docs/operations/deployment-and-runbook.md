# 部署与运行手册

## 部署步骤

1. 在 Vercel 创建项目
2. 配置所有环境变量
3. 部署 Production
4. 确认 `APP_BASE_URL`
5. 调用 `/api/admin/bootstrap-tplink` 初始化消息订阅
6. 验证回调路径是否可访问

## 常见排障

- 次数不足：检查 `service_balance`
- 任务执行失败：检查运行记录与失败表
- 消息不产生：检查不合格结果是否入库、消息规则是否开启
- 图片/录像不可用：检查媒体过期时间与 TP-LINK 抓图/录像前置条件
- 回调未生效：检查生产域名、消息订阅配置与 Vercel 日志

## 运维动作

- 补次数：走站内测试购买或直接写业务表
- 重放回调：保留原始 payload 后重投测试接口
- 刷新任务：调用任务刷新接口手动重跑
