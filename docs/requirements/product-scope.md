# 产品范围与已锁定规则

## 首期目标

- 实现首次引导、标准算法选择、测试购买与次数管理、巡检任务、巡检数据、消息中心。
- 建立一套可持续维护的知识基座，便于后续修复、扩展与联调。

## 范围边界

- `自定义算法`：仅保留入口与说明，不做真实算法管理。
- `服务购买`：不做真实订单校验，站内测试购买成功后直接增加次数。

## 已锁定业务规则

- 初始赠送分析次数：`50000`
- 扣减时机：任务实际执行后
- 扣减单位：`设备数 × 算法数`
- 失败返还次数
- `UNQUALIFIED` 进入消息中心
- `UNAVAILABLE` 忽略
- 关闭任务后仍可查看历史数据和消息
- 趋势、排行、概览均按设备结果条数聚合

## 默认解释

- 异常任务：配置异常或不可执行任务，不包含部分成功任务
- 消息提醒：默认每次巡检不合格即推送
- 购买入口：站内购买弹层与次数不足横幅入口
