# 系统架构

## 总体结构

- `Next.js App Router`
  - 页面渲染与交互
  - Route Handlers 作为轻后端
- `Supabase`
  - 持久化存储
  - 后续可扩展实时订阅与存储能力
- `TP-LINK 商用云开放接口`
  - 算法列表
  - AI 图像检测任务
  - 结果回调
  - 消息订阅
  - 抓图/录像能力

## 轻后端职责

- 保管 AK/SK
- 统一做 TP-LINK 请求签名
- 发起 AI 任务、设置回调地址
- 初始化消息订阅配置
- 接收 TP-LINK 回调并写入数据库
- 对前端暴露业务聚合接口

## 数据流

1. 用户选择标准算法并创建任务
2. 后端设置算法版本并保存任务
3. 定时触发或手动刷新执行任务
4. 成功发起 TP-LINK AI 检测任务后扣减次数
5. 执行失败返还次数
6. AI 回调或轮询结果写入结果表
7. 不合格结果转为消息
8. 前端从业务接口读取任务、统计、消息、媒体数据

## 可扩展点

- 接入真实订单系统替代测试购买
- 迁移内存数据层为 Supabase 全量持久化
- 扩展多企业、多项目、多租户隔离
- 增加 Vercel Cron 以外的调度能力
